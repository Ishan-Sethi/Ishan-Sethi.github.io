Arrays

What is an Array?

An array is a type of data structure in Computer Programming, that holds a collection of data. Different coding languages handle arrays in different ways, but this tutorial will mainly be about how arrays work in Java. Every element in an array must be off the same type, either a primitive data type (char, int, double, etc.), or an object (String, Colors, etc.). When you first declare an array, every value in the array is set to it's default value. Doubles, ints and chars are set to 0, booleans are set to false, and Strings are any objects are set to null. Here is syntax on how to declare, and initialize an array:

int[] intArr; // Declaring a new integer Array
intArr = new int[4]; // Initializes integer array of length 4
String[] objectArr = new String[4]; // Declaring, and initializing an array in the same line

There are two ways to store data in an array, you can set up the entire array when initializing it, or access each individual element and change it. Every value in an array is stored at a particular index, starting at 0 and going to the length of the array minus one. This can be somewhat confusing to new programmers, as the first element in every array is stored at index 0, but we humans normally start counting from 1. Here is an example of how to put data in an array:

String[] names = {"Ishan", "Ms. Krasteva"}; // Automatically sets length to 2
int[] intArr new int[4]; // Initializes integer array of length 4
intArr[0] = 2; // Sets the value at index 0 (first element) to be 2
intArr[1] = -180; // Sets the value at index 1 (second element) to be -180

For more information regarding arrays check out the Java Reference or this GeekForGeeks article

What is an ArrayList?

Arrays can make coding in java alot more effecient when working with a fixed amount of data, but they also have some disadvantages. For one, Arrays aren't flexible length, meaning once you initialize it you cannot resize it to be larger then its original length. This is where an ArrayList comes in. An ArrayList will automatically adjust its size to fit all the elements you must store. You can add, remove and insert elements at any point in an ArrayList. Here is an example on how to use an ArrayList.

import java.util.ArrayList;

ArrayList<Integer> num = new ArrayList<Integer>();
num.add(5); // Adds 5 as the first element
num.add(2); // Adds 2 as the second element
num.add(0, 2); // Inserts 2 at index 0, pushing everything else back
num.set(3, 10); // changes index 3 to be 10
num.remove(0); // removes the element at index 0
// Outputting every int in num
for (int i = 0; i < num.size(); i++) {
    System.out.println(num.get(i));
}

ArrayLists can only store objects, not primitive data types, which is why we used the Integer class instead of int. However, when we add a new element we don't need to create a new Integer() object because Java will automatically cast the int literal to type Integer() (Please Note: this only applies to primitive data types and Strings, not any other objects). To access an elemnt in an ArrayList, we use the ArrayList.get() method and specify which index we want to access, similar to an Array. To insert an item at a specific index, you just have to specify the index in the ArrayList.add() method. ArrayLists are much easier to use, but they also come at a cost. ArrayLists are much more expensive, and take up more memory than regular Arrays.

Why Should we use Arrays/ArrayLists?

When working with basically any quantity of date larger then a single variable, Arrays are always faster to use, and more efficient to design software around. For example, lets say we are programming a video game, and once a player's score reaches 100 points, we reset their score to 0. If we had 100 players, we would need 100 variables to store all of their points, and 100 if statements to check.

if(score1 == 100)
    score1 = 0;
if(score2 == 100)
    score2 = 0;
if(score3 == 100)
    score4 = 0;
// ...
if(score99 == 100)
    score99 = 0;

Using an array to hold thoose scores cuts the code down from 200 lines all the way down to 3:

for (int i = 0; i < scores.length; i++)
    if (scores[i] == 100)
        scores[i] = 0;
